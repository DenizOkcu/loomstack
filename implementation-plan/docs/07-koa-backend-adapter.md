# 07 — Koa Backend Adapter

The Koa adapter provides the HTTP transport runtime for loomstack.

Koa is the v0.1 golden path backend because it is minimal, explicit, and agent-readable.

## Core rule

```txt
Koa handles transport. Product behavior lives in actions, queries, schemas, and policies.
```

## Backend app structure

Generated app backend:

```txt
apps/api/
  package.json
  src/
    server.ts
    context.ts
    routes.generated.ts
```

## `server.ts`

Example:

```ts
import Koa from "koa"
import bodyParser from "koa-bodyparser"
import { registerGeneratedRoutes } from "./routes.generated"

const app = new Koa()

app.use(bodyParser())
registerGeneratedRoutes(app)

app.listen(3001, () => {
  console.log("loomstack API listening on http://localhost:3001")
})
```

## Generated routes

The Koa route file is generated from action and query registries.

```ts
// GENERATED BY loomstack. DO NOT EDIT.

import Router from "@koa/router"
import type Koa from "koa"
import { executeAction, executeQuery } from "@loomstack/runtime"
import { createRequestContext } from "./context"
import { actionRegistry } from "../../../shared/generated/action-registry.generated"
import { queryRegistry } from "../../../shared/generated/query-registry.generated"

export function registerGeneratedRoutes(app: Koa) {
  const router = new Router()

  router.post("/_loomstack/actions/:name", async (ctx) => {
    const name = ctx.params.name
    const action = actionRegistry[name]

    if (!action) {
      ctx.status = 404
      ctx.body = { error: { code: "loomstack4040", message: `Unknown action: ${name}` } }
      return
    }

    const requestContext = await createRequestContext(ctx)
    ctx.body = await executeAction(action, requestContext, ctx.request.body)
  })

  router.post("/_loomstack/queries/:name", async (ctx) => {
    const name = ctx.params.name
    const query = queryRegistry[name]

    if (!query) {
      ctx.status = 404
      ctx.body = { error: { code: "loomstack4041", message: `Unknown query: ${name}` } }
      return
    }

    const requestContext = await createRequestContext(ctx)
    ctx.body = await executeQuery(query, requestContext, ctx.request.body)
  })

  app.use(router.routes())
  app.use(router.allowedMethods())
}
```

## Request context

`context.ts` converts Koa context into framework context.

```ts
import type Koa from "koa"
import { db } from "./db"

export async function createRequestContext(ctx: Koa.Context) {
  return {
    requestId: ctx.get("x-request-id") || crypto.randomUUID(),
    user: await resolveUser(ctx),
    db,
    logger: console
  }
}
```

## RPC endpoint design

v0.1 should use action/query RPC:

```txt
POST /_loomstack/actions/<actionName>
POST /_loomstack/queries/<queryName>
```

Rationale:

- directly maps to action/query files
- avoids REST naming ambiguity
- simpler generated clients
- easier agent inspection

REST can be generated later as an optional public API.

## Error responses

Backend runtime errors should be structured.

Example:

```json
{
  "error": {
    "code": "loomstack3001",
    "message": "Action input validation failed.",
    "action": "createPerson",
    "details": [
      { "path": "name", "message": "Required" }
    ],
    "repair": "Send input matching the action input schema."
  }
}
```

## Koa-specific verifier rules

The verifier should detect:

- manual feature routes added directly to Koa
- stale generated backend routes
- Koa imports inside feature action/query files
- missing generated route registration
- missing body parser in generated template if required

## Manual Koa extensions

For v0.1, allow app-level middleware in `server.ts`, but not feature behavior.

Allowed:

- logging middleware
- CORS middleware
- body parsing
- request ID middleware

Forbidden:

- manual routes implementing feature actions/queries
- direct DB access in route handlers

## Non-goals for v0.1

- GraphQL
- tRPC compatibility
- public REST generation
- subscriptions
- WebSockets
- streaming
- complex auth providers
- multi-tenant request context
