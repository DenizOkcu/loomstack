## App-level file responsibilities

### `loom.config.ts`

Contains project configuration:

```ts
import { defineloomConfig } from "@loom/runtime"

export default defineloomConfig({
  appName: "manager-crm",
  packageManager: "pnpm",
  frontend: "react",
  backend: "koa",
  database: "sqlite",
  featuresDir: "features",
  generatedDir: ".loom"
})
```

### `.loom/context.json`

Machine-readable summary for agents.

Generated by:

```bash
loom context generate
```

### `.loom/graph.json`

Machine-readable feature and dependency graph.

Generated by:

```bash
loom graph generate
```

### `.loom/commands.json`

Machine-readable command registry.

Contains commands agents should use:

```json
{
  "dev": "pnpm dev",
  "test": "pnpm test",
  "typecheck": "pnpm typecheck",
  "verify": "pnpm loom verify"
}
```

### `.loom/generated-files.json`

Manifest of generated files and hashes.

Used by the verifier to detect stale or hand-edited generated files.

## Generated file rules

All generated files must:

1. Start with a generated marker.
2. Be deterministic.
3. Be safe to overwrite.
4. Be listed in `.loom/generated-files.json`.
5. Be excluded from manual business logic.

Marker:

```ts
// GENERATED BY loom. DO NOT EDIT.
```

## Feature file naming rules

Required patterns:

```txt
feature.yaml
AGENTS.md
model.schema.ts
permissions.policy.ts
actions/*.action.ts
queries/*.query.ts
ui/*.view.tsx
ui/*.component.tsx
ui/*.form.tsx
tests/*.test.ts
```

## Dependency direction

Allowed conceptual dependency flow:

```txt
schema -> policy -> action/query -> view
```

Adapter-generated code may import action/query/view registries.

Feature UI may not import backend code or database code.

## Non-goal

Do not support arbitrary app structures in v0.1. loom should enforce the structure it generates.
