# loom framework agent instructions

loom is an agent-operable TypeScript fullstack framework. Product behavior belongs in feature slices; React is presentation; Koa is transport.

## Golden path

- TypeScript and pnpm workspaces
- React + Vite
- Koa
- SQLite
- Zod-backed schemas
- Vitest

## Hard rules

- Prefer explicit contracts and one canonical pattern.
- Generate transport and route wiring; never place behavior in generated files.
- Every generated source file starts with `// GENERATED BY loom. DO NOT EDIT.`.
- Every agent-facing command supports `--json`.
- Errors have stable codes and repair guidance.
- Generated output is deterministic.
- Do not add alternative adapters, plugins, SSR, REST generation, or production auth in v0.1.

## Commands

```bash
pnpm test
pnpm typecheck
pnpm build
pnpm loom --help
pnpm loom --cwd examples/manager-crm verify --json
```

## Package ownership

- `core`: manifests, scanning, graphs, errors, project configuration
- `runtime`: schemas, actions, queries, policies, execution
- `generator`: app/feature scaffolding and all derived files
- `verifier`: architecture and generated-file enforcement
- `react-adapter`: views, hooks, RPC client
- `koa-adapter`: RPC transport
- `sqlite-adapter`: v0.1 persistence
- `cli`: agent-facing command surface
